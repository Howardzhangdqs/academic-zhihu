# 知乎过滤器 项目介绍

## 起因

知乎曾经是国内最优质的问答社区之一，尤其是理工科领域，有大量从业者和研究者在上面分享经验、讨论技术细节。但不知道从什么时候开始，我的首页推荐流里充斥着情感问题、社会新闻、键盘政治和各种营销号内容。我并不是说这些内容没有价值，但作为一个学生/科研狗，我打开知乎的目的是想看到跟学术和技术相关的讨论，而不是又一次被推送"如何看待某某事件"。

我试过关注列表、屏蔽关键词，效果都不理想。关注列表信息量太小；屏蔽关键词治标不治本，总有漏网之鱼，而且还会误伤正常内容。

后来我想，既然现在大语言模型的能力已经这么强了，能不能让它来帮我判断一条知乎内容是不是学术内容？于是就有了这个项目。

## 它是什么

简单来说这是一个油猴脚本（Tampermonkey / Greasemonkey 用户脚本），安装之后打开知乎首页，它会自动分析你首页上的每一条问题，用你提供的大语言模型 API 判断这条内容是否属于学术范畴，然后把非学术内容过滤掉（你可以自定义过滤提示词）。

过滤的方式有三种，你可以自由选择：第一种是"颜色变淡"，非学术内容会变成半透明的灰色，鼠标悬停时恢复正常，适合你想快速浏览但又不想完全丢失信息的场景；第二种是"折叠"，非学术内容会被折叠成一个小条，鼠标悬停时展开，适合你只想看到学术内容的场景；第三种是"折叠加变淡"，两者同时生效，过滤效果最强。

## 核心功能

第一个功能是智能分类。脚本会监听知乎首页的信息流，每当有新的内容加载出来，就会把标题提取出来，打包发送给你配置的大语言模型 API。模型会根据你设定的判断标准，返回每条内容是否属于学术内容。当一条内容正在被识别，它的左侧会出现一条灰色的边框。整个过程是自动且实时的，你不需要做任何额外操作，正常刷知乎就行。

第二个功能是自定义提示词。这是我认为这个脚本最有价值的地方。每个人对"学术内容"的定义可能不一样，一个做计算机科学的人和一个做生物学的人，他们想看到的内容大概率是不同的。所以脚本允许你编辑系统提示词，你可以精确地定义什么算学术内容、什么不算。比如默认的提示词会过滤掉中医相关的内容，但如果你恰好对中医现代化研究感兴趣，你可以把这条规则改掉。你还可以加入更多规则，比如"只保留与机器学习相关的内容"或者"过滤掉所有生活类问题"。提示词完全由你控制，模型只是执行你的判断标准。

第三个功能是多模型支持。脚本支持两种 API 接口格式：OpenAI 兼容格式和 Anthropic 格式（不知道为什么感觉 Anthropic 格式目前还有点问题）。这意味着你可以使用 OpenAI 的 GPT 系列模型，也可以使用 Anthropic 的 Claude 系列模型，还可以使用任何兼容 OpenAI 接口的第三方服务，比如 DeepSeek、智谱 GLM、OpenRouter 等等。你只需要在设置里填入对应的 Base URL、API Key 和模型名称就可以了。为了兼容各种 API 代理和特殊需求，脚本还提供了自定义请求头和自定义请求体的选项，你可以用 JSON 格式添加任意额外的参数。

第四个功能是连通性测试。配置好 API 之后，你可以点击"测试连通"按钮，脚本会发送一个简单的测试请求到你的 API，确认配置是否正确、网络是否通畅、模型是否可用，避免配置有误导致脚本静默失败。

## 实现细节

这个脚本是用 Vue 3 写的，通过 vite-plugin-monkey 打包成油猴脚本。它会通过 MutationObserver 监听知乎首页推荐区域（`.Topstory-recommend`）的 DOM 变化，每当有新的内容卡片出现，就会提取其中的标题、摘要、作者等信息，然后按照你设置的批量大小分批发送给模型进行分类。分类结果返回后，脚本会根据你选择的隐藏模式，给对应的 DOM 元素添加 CSS 类名来实现过滤效果。所有的 API 请求都是通过油猴的 `GM_xmlhttpRequest` 发送的，这样可以绕过浏览器的跨域限制。你的 API Key 和所有设置都保存在油猴的本地存储中，不会上传到任何地方。

## 使用方法

使用这个脚本的前提是你有一个可用的大语言模型 API。如果你已经有 OpenAI、Anthropic、DeepSeek 或者其他模型的 API Key，那就可以直接开始了。

首先，确保你安装了 Tampermonkey 或其他什么能运行脚本的浏览器扩展。然后安装这个脚本。打开知乎首页，你会看到左下角多了一个悬浮按钮，点击它打开设置面板。

在设置面板里，选择你的 API 提供商，填入 Base URL、API Key 和模型名称。如果你用的是 OpenAI 或者兼容 OpenAI 接口的服务，选"OpenAI Compatible"；如果用的是 Anthropic 的 Claude，选"Anthropic"。

配置好 API 之后，建议先点一下"测试连通"按钮，确认能正常调用。然后你可以根据需要编辑系统提示词、选择隐藏模式、调整批量大小（批量越大，API 调用次数越少，但单次请求的 token 消耗越大）。最后点击保存，脚本就会开始工作了。

## 关于隐私

这个脚本完全开源，使用不会收集任何用户数据，不会向除了你配置的 LLM API 之外的任何服务器发送请求。你的 API Key 存储在浏览器本地，脚本发送给模型的仅仅是知乎信息流中公开可见的标题和摘要文本，不包含你的个人信息、浏览历史或任何其他敏感数据。
